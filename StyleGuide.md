# Markdown Style Guide 📘

A practical reference for creating clean, readable, and professional Markdown documents.

---

## ✨ Headings

Use headings to organize content clearly.

- `#` for the main title
- `##` for major sections
- `###` for subsections
- `####` for smaller grouped points if needed

Example:

```md
# Title
## Section
### Subsection
```

---

## 📝 Text Formatting

Use formatting sparingly for readability.

- **Bold** with `**text**`
- *Italic* with `*text*`
- ~~Strikethrough~~ with `~~text~~`
- `Inline code` with backticks
- Links with `[text](https://example.com)`

---

## 📋 Lists

### Bulleted List

```md
- Item 1
- Item 2
  - Sub-item
```

### Numbered List

```md
1. First
2. Second
3. Third
```

### Nested Lists

```md
- Parent item
  - Child item
    - Grandchild item
```

---

## ✅ Checklists

```md
- [x] Completed task
- [ ] Pending task
```

---

## 💡 Blockquotes

Use blockquotes to highlight notes, warnings, or tips.

```md
> This is an important note.
> It can span multiple lines.
```

---

## 🔥 Callouts

```md
> 💡 Tip: Keep your wording clear and concise.
> ⚠️ Warning: Always save your work before major edits.
> ✅ Success: Your documentation is now easier to read.
```

---

## 🖼️ Images

```md
![Alt Text](https://example.com/image.png)
```

For resizing in GitHub or documentation platforms, HTML can also be used:

```html
<img src="https://example.com/image.png" width="300" />
```

---

## 📊 Tables

Tables are useful for comparing features, statuses, or options.

```md
| Feature | Description | Status |
|--------|-------------|--------|
| Headings | Organize sections | ✅ |
| Lists | Bullets and numbers | ✅ |
| Images | Add visuals | ✅ |
```

---

## 💻 Code Blocks

Use fenced code blocks for commands, configuration, or code examples.

```python
def greet(name):
    return f"Hello, {name}!"
```

You can also label the language for better highlighting:

```bash
git clone https://github.com/username/project.git
cd project
```

---

## 🔗 Links and References

Good documentation should link to related files and resources when helpful.

```md
- [Project README](README.md)
- [Style Guide](StyleGuide.md)
- [Official GitHub Docs](https://docs.github.com)
```

---

## 🎨 Styling Tips

- Use emojis sparingly for visual clarity
- Add horizontal rules `---` to separate sections
- Keep paragraphs short and scannable
- Preview your Markdown before sharing it
- Use consistent naming and punctuation throughout the document

Example:

```md
## 📌 Project Status
- 🚀 Feature A — Done
- 🛠️ Feature B — In Progress
- ⏳ Feature C — Pending
```

---

## ⚠️ Common Mistakes to Avoid

- Overusing emojis and decorative elements
- Mixing too many heading levels without reason
- Writing long paragraphs with no structure
- Forgetting to preview the final Markdown output

---

## 🧭 Final Tips

- Keep content structured: heading, explanation, example
- Mix text, lists, and visuals for better readability
- Use consistent formatting throughout the document
- Always review the final result before publishing
- Make the content useful for both beginners and experienced readers
